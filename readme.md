# 자기소개서

### 주니어 개발자로서 목표
```
 현직 개발자로부터 유의미한 데이터를 만들기 위한 공정 과정에서 이미지로부터 텍스트를 추출하는 python code를 
제공 받았던 경험, 상품 페이지에서 고객의 개선 요구 사항 등을 수렴하기 위한 기능 구현 프로젝트에서 실무자와 
소통한 경험 등을 통해 `무`에서 `유`를 만들고 유연한 사고로 보다 효율적인 방법을 찾아내는 개발 업무에 
큰 매력을 느끼게 되었습니다. 금전적인 이유로 도전하지 못했으나 현재 국비지원을 통해 정보처리 산업기사 과정을 
수강하고 있습니다.
 처음 개발자를 목표로 세운지 6년만에 목표 달성을 가시권에 두고 있습니다. 지금 이 기회가 현실적으로 잡을 수 
있는 마지막 기회인 만큼 롱런하는 개발자가 되기 위해 기본기를 탄탄하게 다지겠습니다. 입사 후 3년 이내에 
아래 두 가지 목표 우선 달성하겠습니다.
 첫 번째, 코드 리뷰를 잘 하는 개발자가 되겠습니다. 입사 후 기회가 주어진다면 선배님들의 코드를 많이 
리뷰하겠습니다. 머릿속으로 먼저 돌려볼 수 있다는 것은 그만큼 기본기가 잘 잡혀 있다고 볼 수 있으며, 코드를 
읽어보는 것으로도 작동의 유무나 발생 가능한 문제나 개선안 등을 먼저 제시할 수 있는 개발자가 되겠습니다.
 또 한가지의 목표는 잊을만 하면 찾는 후임이 되고자 합니다. 주니어 개발자로서 정답을 찾는데 한계가 있는것이 
분명하고, 상급자를 자주 찾게 됩니다. 다만, 질문자로서 스스로 묻는것이 무엇인지 알고 조언을 구하는 사전 지식을 
준비하는 자세나 정답을 찾았으나 이게 정답인지 모르는 경험의 부재는 답변자가 중요하게 보는 덕목이 될 것이라고 
생각합니다. 본 지원자는 적어도 스스로가 묻는것에 대한 이해도를 가지고 상급자를 찾아가는 주니어 개발자가 되고자 
하며, 스스로 지식을 쌓는 기간을 `잊을만`으로 정하여 가끔 `찾아올 때가 된` 사원으로 첫 이미지를 만들겠습니다.
```

### 이전 직장에서의 업무 경험

```
ㅁ 동일 상품 구분을 위한 시스템 학습 데이터 생성
쿠팡의 PBS 조직 내 Automation팀에서 상품 중복 제거의 자동화를 위해 쿠팡에 입점한 파트너사들이 등록하는 상품들과 
기존에 등록되어 판매중인 상품들의 동일 여부를 판단한 데이터를 쌓고, 누적된 데이터를 이용해 시스템을 학습시키는 
과정에 참여했습니다.
입점 판매자의 상품 등록에 앞서 동일한 상품을 하나의 상품 페이지에 노출하는 쿠팡의 고객 쇼핑 경험 개선을 위한 
업무로, 로직으로 판별하지 못하는 상품 정보를 확인하여 중복 제거하기 위한 데이터를 제공했습니다.
아이템 마켓 특성 상 동일 상품을 하나의 상품 페이지에서 노출하는데, 각 판매자는 상품과 관련 없는 정보를 상품 정보에 
포함하는 방법으로 동일 상품을 판별하는 로직으로부터 벗어났습니다. 이와 같은 케이스를 잡기 위해 불필요한 상품 정보를 
필터링 할 수 있는 패턴이나 상품군 별로 불필요한 기준을 수정하는 등의 업무를 통해 machine learning을 위한 신뢰할 
수 있는 데이터를 제공하는 업무를 수행했습니다.

ㅁ 업무 자동화를 위한 노력
오프라인 유통 기반의 중소 업체에 재직하며 다기능 ERP `E-COUNT`를 높은 수준으로 사용했습니다. 주로 생산이나 
가공/반가공 업체에서 사용하기에 유용한 프로그램인데, 오프라인 유통/온라인 판매를 위한 재고관리 프로그램으로 
유용하게 변형하였습니다. 구글 스프레드 시트로 각 쇼핑몰 (쿠팡, 위메프, 스마트스토어 등)에서 수집한 정보를 입력하여 
대한통운 택배 시스템으로부터 송장번호를 부여받을 수 있도록 데이터를 가공했으며, 다수의 쇼핑몰에서 수집한 주문 
정보를 E-COUNT에 업로드할 수 있는 데이터로 2차 가공했습니다. 시트에서 두가지의 데이터를 가공하여 재고관리에도 
활용할 수 있었고, 오출고로 인한 손실 절감에 기여할 수 있었습니다.
```

### 프로젝트 진행 간 역할과 비중

```
웹 어플리케이션 프로젝트로 `project05_JejuMatjip`을 진행했습니다. STS 3.9.18을 IDE로 채택했으며, 
Spring Framework 3.1.1이 사용됐습니다.
 강의 내 강사님의 주도로 팀이 나뉘어졌고, 팀 별 프로젝트 주제 선정과 구조 설계 등을 위한 미팅에는 장례로 인해 
참석하지 못했습니다. 초기 미팅에서 본 지원자는 팀 리더로 선정되었으며, 프로젝트의 초기 진행에는 리더가 없었음에도 
팀원들간의 협의로 기본 틀을 설정 한 점을 공유 받았습니다. 그러나, 강의에 복귀 후 진행 현황을 살펴보니 각 구성원이 
코딩한 class와 bean, table 구조 등이 상이하여 진행에 문제가 발생하고 있던 점을 확인했습니다. 프로젝트의 ETA는 14일이었고, 
3일이 지난 후 확인 된 진행 상황을 빠르게 수습하지 않으면 마감 기한을 지키지 못할 수 있었던 상황이었습니다.
 빠른 의사 결정이 필요하다고 판단했고 즉시 팀원 전원을 소집해 각자의 개인별 작업 현황 브리핑을 요청하고 팀원 전체가 서로의 
브리핑을 들을 수 있도록 했습니다. 브리핑을 통해 서로가 다른 방향으로 진행하고 있다는 것을 인지 시켰으며, 리더가 결정한 
진행 방향과 타당한 사유를 팀원들에게 공지했습니다. 파트별 담당자들은 본 지원자의 의사 결정 중 일부 개선안을 제시하였고, 
대부분의 개선 의견은 반영해 진행하기로 협의하였습니다.
 복귀 후 팀에 기여한 부분은 아래와 같습니다. 각 팀원 간 기술적으로 큰 격차는 없었지만, 팀원들에게 막히는 부분에 도움을 
줄 수 있는 수준에 있는 본 지원자는 주어진 파트 작업을 사전에 완료하고 각 파트에서 요청하는 요청 사항을 서포트 했습니다.
프로젝트 진행 중 최초로 설정한 틀을 벗어나는 변경 사항에 대해 의사 결정하기 전 각 팀원의 의견 수렴이나 
업무 현황 파악 / 공유를 위해 하루 2번 약식으로 미팅을 진행했습니다.
```
